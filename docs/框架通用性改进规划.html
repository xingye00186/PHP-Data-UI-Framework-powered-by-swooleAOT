框架通用性改进规划

让 vue-calc 框架从"calculator 专属"演变为"多应用可复用"的通用框架

1. 概述

目标：让第二个应用（如 todo-app）能以最小修改复用 vue-calc 框架，无需修改 Application、BaseRenderer 或 main.php。

当前 vue-calc 框架在 calculator 应用中运行良好，但其中存在大量硬编码、隐式约定和重复逻辑，导致创建新应用时必须深入修改框架核心文件。本文档系统梳理这些反模式，并提出分阶段改进方案。

3. P1 改进项（立即实施，~200行改动）

3.1 窗口配置参数化

问题：窗口标题、尺寸在 Application.php 和 main.php 中硬编码。

方案：

  • SFC 编译器在 AppLayout_gen.php 中生成 getApplicationConfig()
  • 返回 ['windowTitle' => ..., 'windowWidth' => WINDOW_WIDTH, 'windowHeight' => WINDOW_HEIGHT]
  • Application.initWindow() 读取该函数而非硬编码

生成代码示例：

function getApplicationConfig(): array {
    return [
        'windowTitle'  => 'TodoApp - SFC Data-Driven App',
        'windowWidth'  => WINDOW_WIDTH,
        'windowHeight' => WINDOW_HEIGHT,
    ];
}

3.2 事件常量集中化

问题：WM_LBUTTONDOWN、SW_SHOW 等常量在每个应用的 main.php 中重复定义。

方案：

  • 创建 framework/WinConstants.php 集中定义 SW_SHOW, WM_LBUTTONDOWN, WM_QUIT 等
  • main.php 改为 require __DIR__ . '/../../framework/WinConstants.php';

WinConstants.php 示例：

<?php
// framework/WinConstants.php — Windows 消息与显示常量
const SW_SHOW            = 5;
const WM_LBUTTONDOWN    = 0x0201;
const WM_LBUTTONUP      = 0x0202;
const WM_MOUSEMOVE      = 0x0200;
const WM_QUIT           = 0x0012;
const WM_DESTROY        = 0x0002;
const WM_PAINT          = 0x000F;
const WS_OVERLAPPEDWINDOW = 0x00CF0000;
const CW_USEDEFAULT     = -2147483648;

3.3 attachLayout 接口简化

问题：attachLayout(string $name, int $layoutIdx) 的第二个参数无实际用途。

方案：

  • 去掉 attachLayout 的第二个参数，签名改为 attachLayout(string $name)
  • $activeLayouts 改为纯名称列表 string[]
  • Application 改为简单循环注册

修改前：

foreach ($layoutNames as $i => $name) {
    $app->attachLayout($name, $i + 1);
}

修改后：

foreach ($layoutNames as $name) {
    $app->attachLayout($name);
}

4. P2 改进项（短期）

4.1 文本排版外部化

问题：BaseRenderer 中文本字号缩放逻辑（textLen > 12 则 24px）为 calculator 专属策略。

方案：

  • 元素数据中添加 autoResize 字段控制是否自动缩放
  • BaseRenderer 提供 protected autoResizeFont() 方法，应用可覆写

数据格式扩展：

[
    'type'       => 'element',
    'text'       => '0',
    'autoResize' => true,   // 新字段：是否自动缩放字号
    'rect'       => [0, 0, 100, 40],
]

BaseRenderer 默认实现：

protected function autoResizeFont(string $text, float $baseSize, array $rect): float {
    if (strlen($text) > 12) {
        return 24.0;
    }
    return $baseSize;
}

4.2 按钮样式配置化

问题：按钮文字样式硬编码（22pt/居中/加粗），新应用无法自定义。

方案：

  • 按钮数据结构添加 labelFontSize, labelAlign, labelColor, labelBold
  • BaseRenderer 读取字段而非硬编码

数据格式扩展：

[
    'type'          => 'button',
    'text'          => '7',
    'labelFontSize' => 22,     // 新字段：字号（pt）
    'labelAlign'    => 'center', // 新字段：对齐方式
    'labelColor'    => '#ffffff', // 新字段：文字颜色
    'labelBold'     => true,     // 新字段：是否加粗
    'rect'          => [0, 0, 80, 60],
]

4.3 布局收集逻辑提取

问题：render() 和 handleClick() 的分层遍历逻辑在 BaseRenderer 和 Application 中重复实现。

方案：

  • 提取 gatherLayout() 和 getMaxLayer() 为 BaseRenderer 公共方法
  • Application.handleClick() 调用这些方法而非重复实现

提取方法签名：

public function gatherLayout(string $layoutName): array
public function getMaxLayer(array $layout): int

5. P3 改进项（长期架构）

5.1 Application 基类化

问题：每个新应用都需要完整实现 Application 类，无法复用启动流程。

方案：

  • 创建 framework/ApplicationBase.php 抽象类
  • 定义 createComponent(), getApplicationConfig() 抽象方法
  • 提供 bootstrap() 模板方法标准化启动流程

ApplicationBase 骨架：

abstract class ApplicationBase {
    abstract protected function createComponent(string $class): ReactiveComponent;
    abstract protected function getApplicationConfig(): array;

    public function bootstrap(): void {
        $config = $this->getApplicationConfig();
        $this->initWindow($config);
        $this->registerComponents();
        $this->runMessageLoop();
    }
}

5.2 布局数据格式规范

问题：element/button/condition 数据格式完全隐式约定，无文档。格式变更导致无声失败。

方案：

  • 创建 framework/LAYOUT_DATA_FORMAT.md
  • 显式定义 element/button/condition 所有字段、类型、默认值

规范示例（element）：

| 字段       | 类型    | 必填 | 默认值  | 说明              |
|-----------|---------|------|---------|-------------------|
| type      | string  | 是   | -       | 固定值 "element"  |
| text      | string  | 是   | -       | 显示文本          |
| rect      | int[4]  | 是   | -       | [x,y,w,h]        |
| color     | string  | 否   | "#fff"  | 背景色            |
| autoResize| bool    | 否   | false   | 是否自动缩放字号  |

6. 验收标准

目标验收场景：新应用 todo-app 启动流程：编写 .vue 文件 → 运行 SFC 编译 → AOT 编译 → 运行。无需修改 Application、BaseRenderer 或 main.php 的框架逻辑。

具体验收检查项：

  • ✅ todo-app 的窗口标题由编译器生成，无需手动修改 Application
  • ✅ todo-app 的 main.php 仅 require 框架常量文件，无重复定义
  • ✅ todo-app 可通过数据字段自定义按钮样式，无需修改 BaseRenderer
  • ✅ todo-app 可通过 autoResize 控制文本缩放行为
  • ✅ 布局段名称拼写错误在编译期报告而非静默失败
