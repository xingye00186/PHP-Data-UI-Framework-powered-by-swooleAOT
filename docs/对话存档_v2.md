### 7.2 正则 Bug 调试

**Bug 1：样式颜色正则冲突**

`preg_match('#background...#RRGGBB#', $body)` — `#` 同时是分隔符和 CSS 颜色前缀。

```
preg_match('#background\s*:\s*#([0-9a-fA-F]{6})#', ...)
            ↑ 分隔符          ↑ 被当成结束分隔符！
```

**修复**：改用 `~` 分隔符。

**Bug 2：文本元素未解析**

`<text :bind="expression" class="expr-text" ...>` — 属性名 `:bind` 含冒号，但正则字符类 `[a-z-]+` 不含 `:`。

**修复**：改为 `[a-z:@-]+`。

### 7.3 AOT 兼容性修复

**Bug 3：文件名点号 → C++ 编译错误**

`CalculatorLayout.gen.php` → AOT 生成 C++ 函数名 `register_CalculatorLayout.gen_symbols` → `.gen` 在 C++ 中非法。

```
error C3927: "->": 非函数声明后多余的返回类型
error C3484: 语法错误: 应在 "->" 前为 "("
```

**修复**：文件名改为 `CalculatorLayout_gen.php`（下划线）。

**Bug 4：const 复杂数组不被 AOT 支持**

`var_export` 生成的嵌套 `const LAYOUT = [...]` 被 AOT 编译器跳过。

**修复**：改为 `function getLayout(): array { return [...]; }`。
