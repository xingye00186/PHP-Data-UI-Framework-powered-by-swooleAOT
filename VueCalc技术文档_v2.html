一、项目概述

VueCalc v2 实现了完整 SFC 编译器管道：开发者编写 .vue 单文件组件 → PHP SFC 编译器将其编译为 PHP ReactiveComponent 子类 + 布局数据 → AOT 编译器生成原生 exe。

核心理念：PHP 替代 JS 实现数据与业务逻辑，C++ (GDI) 渲染界面。UI 渲染完全由数据驱动——UI = f(state)。

关键 AOT 约束：Swoole Compiler AOT 不支持 __get/__set 魔术方法、反射、eval。因此采用直接属性声明 + 手动脏标记 (dirty flag) 模式。

2.2 渲染层 → v2: 遍历 LAYOUT 数据

public function render(): void
{
    $hdc = vue_begin_paint($this->hWnd);
    $layout = getLayout();

    // 渲染元素（rect 背景 + text 文本）
    foreach ($layout['elements'] as $el) {
        if ($el['type'] === 'rect') {
            vue_fill_rect($hdc, $el['x'], $el['y'], $el['w'], $el['h'], $el['color']);
        } elseif ($el['type'] === 'text') {
            // 读取组件属性 → 计算坐标 → 绘制
            $text = $this->getBindValue($el['bind']);  // 如 'display'
            $this->renderTextElement($hdc, $el, $text);
        }
    }

    // 渲染按钮（背景 + 边框 + 居中文字）
    foreach ($layout['buttons'] as $btn) {
        vue_draw_button($hdc, $btn['x'], $btn['y'], $btn['w'], $btn['h'], $btn['bg'], $btn['border']);
        // 居中绘制 label 文字...
    }

    vue_end_paint($this->hWnd, $hdc);
}

8.3 SFC 编译器关键实现细节

CSS 颜色映射

// CSS:  .btn-op { background: #ff9500; color: #ffffff; }
// GDI:  BGR integer (Win32 COLORREF format)

function hexToBgr(string $hex): int {
    $r = hexdec(substr($hex, 0, 2));
    $g = hexdec(substr($hex, 2, 2));
    $b = hexdec(substr($hex, 4, 2));
    return ($b << 16) | ($g << 8) | $r;  // 0x0095FF
}

function borderColor(int $bg): int {
    // 每个通道 +20，自动推导边框色
    $r = min(255, (($bg >> 16) & 0xFF) + 20);
    $g = min(255, (($bg >> 8)  & 0xFF) + 20);
    $b = min(255, ($bg         & 0xFF) + 20);
    return ($r << 16) | ($g << 8) | $b;
}

Grid 布局编译期计算

9.3 AOT 兼容性注意事项

SFC 编译器设计必须遵循的 AOT 约束：
  • 文件名不含点号：Layout.gen.php → Layout_gen.php（否则生成无效 C++ 函数名）
  • 不用 const 复杂数组：改为 function getLayout(): array
  • 不用变量属性：$obj->$variable → 改为显式 if/else 映射
  • 不用变量方法调用：$obj->$method() → 改为显式 if/else 路由
  • 编译器运行在 AOT 之前：sfc-compiler.php 是标准 PHP CLI 脚本，不经过 AOT 编译
